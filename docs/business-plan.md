## 1. 执行摘要

**日本旅游地图（Japan Otaku Map）** 是一款专为中文用户打造的日本动漫周边店铺交互式地图平台。产品精准解决了一个长期痛点：中国动漫爱好者赴日旅行时，缺乏一个系统化、可视化、中文友好的动漫店铺导航工具。

### 核心数据

| 指标 | 数值 |
|------|------|
| 收录店铺总数 | **176 家** |
| 覆盖连锁品牌 | **7 大品牌** |
| 地理覆盖范围 | **全日本（北海道至冲绳）** |
| MVP 开发周期 | **3 天** |
| 当前托管成本 | **¥0（GitHub Pages 免费托管）** |

### 七大连锁品牌覆盖

| 品牌 | 店铺数 | 标识颜色 |
|------|--------|----------|
| Animate（アニメイト） | 48 家 | 🩷 粉色 |
| Suruga-ya（骏河屋） | 37 家 | 🔵 蓝色 |
| Lashinbang（らしんばん） | 30 家 | 🟣 紫色 |
| Melonbooks（メロンブックス） | 29 家 | 🟢 绿色 |
| GAMERS（ゲーマーズ） | 14 家 | 🟡 黄色 |
| Mandarake（まんだらけ） | 11 家 | 🟠 橙色 |
| K-Books（ケーブックス） | 7 家 | 🔴 暗红色 |

### 商业机会

中国赴日旅游市场正在经历后疫情时代的强劲复苏。2024年中国赴日游客预计恢复至约700万人次，其中30%-40%对动漫文化有浓厚兴趣。然而，目前市场上没有一款产品能系统性地满足这一群体"在日本找动漫店"的核心需求。

日本旅游地图以**零成本 MVP** 验证了产品-市场匹配度，凭借**3天极速迭代**的执行力，从87家店铺快速扩展至176家，覆盖7大核心连锁品牌。产品已上线 Web 端和 Android APK，具备搜索、筛选、离线缓存等核心功能。

我们正在寻求 **¥500,000 种子轮融资**，用于产品功能扩展、市场推广和数据积累，目标在12-18个月内达到 **10万 MAU**，建立中文动漫旅行工具的绝对领先地位。

## 2. 公司概况

### 使命

**让中国动漫爱好者在日本的购物体验变得轻松、高效、愉快。**

每年数百万中国动漫迷奔赴日本，面对语言障碍和信息碎片化，找到心仪的动漫周边店铺往往依赖小红书零散攻略或低效的地图搜索。我们的使命是将这些碎片化信息整合为一个直观、实时、全面的地图平台，成为每位赴日动漫爱好者的必备工具。

### 愿景

**成为中文用户在日本乃至全球动漫文化旅行的第一入口。**

从日本出发，逐步覆盖韩国、台湾地区及东南亚市场，打造一个跨越国界的动漫文化旅行生态平台。

### 当前状态

- **产品阶段：** MVP 已上线，可通过 Web 和 Android APK 访问
- **用户增长：** 有机增长阶段，产品链接已在社交平台自然传播
- **技术成熟度：** 核心功能完整，性能经过优化（WebGL 原生标记点，零卡顿）
- **数据积累：** 176家店铺数据已录入，含中文描述、地理坐标、分类标签

### 法律结构

> 公司法律实体待设立。计划注册为科技有限公司，考虑在中国大陆或香港设立主体，以便于后续融资和跨境业务拓展。

## 3. 市场分析

### 3.1 行业背景

#### 中国赴日旅游市场

中国一直是日本最大的入境旅游客源国之一。根据日本国家旅游局（JNTO）公开数据：

| 年份 | 中国赴日游客数 | 备注 |
|------|---------------|------|
| 2019 | **959万人** | 历史峰值 |
| 2020-2022 | 大幅下降 | COVID-19 疫情影响 |
| 2023 | **~240万人** | 复苏起步（受签证放开时间限制） |
| 2024 | **~700万人（估）** | 强劲反弹，签证政策持续放宽 |
| 2025-2026 | **预计恢复至800-950万** | 接近疫前水平 |

中国游客以**消费力强**著称——2019年中国游客在日消费总额约**1.77万亿日元**，人均消费位居各国游客之首。动漫周边、手办、同人志等文化商品是重要消费品类。

#### 动漫旅游——从亚文化走向主流

- **"圣地巡礼"（聖地巡礼）** 已成为日本旅游的核心吸引力之一。游客会专程前往动漫作品的取景地和相关店铺
- 日本动画协会（AJA）2023年报告显示，日本动画产业规模达到 **2.93万亿日元**，连续多年创历史新高
- 中国是日本动漫海外市场的最大消费国，B站（Bilibili）动漫活跃用户超过**1.8亿**
- 赴日游客中对动漫文化感兴趣的比例持续提升，多项调查显示"动漫/文化体验"已成为中国游客赴日旅行的**前三大动机**之一

#### 消费者行为趋势

- **"目的性消费"增强：** 游客出发前在小红书、B站做大量功课，需要精准的店铺位置信息
- **"体验式旅行"兴起：** 不再满足于东京银座购物，追求"找到宝藏店铺"的独特体验
- **"工具化需求"明确：** 用户需要一个"打开就能用"的地图工具，而非逐篇翻阅旅行攻略

### 3.2 目标用户画像

#### 核心用户（Primary）

| 维度 | 描述 |
|------|------|
| **年龄** | 18-35岁 |
| **身份** | 中国大陆动漫爱好者 |
| **场景** | 计划赴日旅行，或已在日本旅行中 |
| **行为** | 出发前在小红书/B站查攻略 → 旅行中需要实时导航 → 回国后分享购物战果 |
| **痛点** | 信息碎片化、语言障碍、Google Maps 无法按"动漫店"精准筛选 |
| **付费意愿** | 对真正好用的工具愿意付费（对标小红书薯队长、航旅纵横PRO等） |

#### 次要用户（Secondary）

- **在日华人：** 长期在日本生活和工作的中文用户，有持续购买动漫周边的需求
- **回头客：** 多次赴日旅行的深度动漫爱好者，追求发现新店铺
- **代购群体：** 为国内客户代购动漫周边的个人或小团队

#### 潜在用户（Tertiary）

- **日本本地动漫爱好者：** 未来中→日本地化后可覆盖
- **其他亚洲动漫爱好者：** 韩国、东南亚用户群体

#### 用户旅程

```
发现需求          →    搜索信息         →    使用工具          →    消费行动          →    分享推荐
"日本哪里买手办"    小红书/B站/知乎      打开日本旅游地图      前往店铺购买         发小红书/朋友圈
                  攻略零散，信息过时    搜索/筛选/定位        体验满意            自然口碑传播
```

### 3.3 市场规模估算

我们采用 **TAM-SAM-SOM** 模型进行市场规模测算：

#### TAM（总可触达市场）

- **定义：** 所有中国赴日游客
- **规模：** 年均 **700万-1000万人次**（后疫情恢复期）
- **依据：** JNTO官方数据，2019年峰值959万，预计2025-2026年恢复至800-950万

#### SAM（可服务市场）

- **定义：** 对动漫文化感兴趣的中国赴日游客
- **占比：** 约 **30%-40%**
- **规模：** **200万-400万人次/年**
- **依据：** 多项旅游调查显示"动漫/流行文化"是中国游客赴日旅行的前三大兴趣领域；B站用户渗透率持续提升

#### SOM（可获取市场）

- **定义：** 实际使用我们地图产品的用户
- **保守估算：** SAM的 **2%-5%**
- **规模：** **4万-20万用户/首年**
- **依据：** 参考同类垂直工具（如极简汇率、旅行翻译官等）首年用户规模

#### 收入潜力

| 场景 | MAU | ARPU（年） | 年收入 |
|------|-----|-----------|--------|
| 保守 | 4万 | ¥10 | ¥40万 |
| 基准 | 10万 | ¥15 | ¥150万 |
| 乐观 | 20万 | ¥20 | ¥400万 |

> 即使在最保守的场景下，仅凭少量广告和基础付费功能，首年收入也可覆盖运营成本。随着用户规模增长和商业化深入，收入增长潜力巨大。

## 4. 产品与服务

### 4.1 产品功能详述

日本旅游地图当前版本（v2.8）已具备以下核心功能：

#### 地图引擎

- **矢量瓦片地图：** 基于 MapLibre GL JS 5 和 OpenFreeMap 矢量瓦片，流畅的缩放和平移体验
- **WebGL 原生标记点：** 176个店铺标记以 WebGL 原生渲染，实现零延迟、零卡顿的地图交互
- **7色品牌分类：** 每个连锁品牌使用独立颜色标识，地图上一目了然

| 品牌 | 颜色 | 店铺数 |
|------|------|--------|
| Animate（アニメイト） | 粉色 | 48 |
| Suruga-ya（骏河屋） | 蓝色 | 37 |
| Lashinbang（らしんばん） | 紫色 | 30 |
| Melonbooks（メロンブックス） | 绿色 | 29 |
| GAMERS（ゲーマーズ） | 黄色 | 14 |
| Mandarake（まんだらけ） | 橙色 | 11 |
| K-Books（ケーブックス） | 暗红色 | 7 |

#### 搜索与筛选

- **全文搜索：** 支持按店铺名称、描述、标签、地址搜索
- **地区筛选：** 按都道府县（都道府県）过滤，快速定位目标区域
- **品牌筛选：** 支持任意组合勾选7大品牌进行过滤
- **多维排序：** 支持按评分、名称、录入日期、热度等维度排序

#### 交互体验

- **响应式布局：** 桌面端侧边栏 + 移动端抽屉式面板，适配所有屏幕尺寸
- **卡片-标记联动：** 鼠标悬停店铺卡片时，地图上对应标记高亮，反之亦然
- **弹出详情窗：** 点击标记显示店铺详细信息（名称、地址、分类、描述）

#### 离线与移动端

- **Service Worker 离线缓存：** 支持弱网/离线环境下使用（旅行中地铁等场景）
- **Android APK：** 已提供可安装的 Android 应用（WebView 封装），支持离线使用
- **PWA 就绪：** 可添加到手机桌面快捷访问

#### 部署与性能

- **GitHub Pages 自动部署：** 代码推送即自动构建部署，零运维成本
- **代码分割：** 3个chunk按需加载，首屏加载极速
- **SEO 优化：** 结构化数据、语义化HTML、meta标签完整

### 5.2 竞争优势（竞争壁垒）

#### 1. 数据壁垒

**176家动漫专业店铺的结构化中文数据**——目前市场上没有任何产品拥有如此全面的、面向中文用户的日本动漫店铺数据库。每家店铺包含精准地理坐标、中文描述、品牌分类和详细标签。这个数据集是通过大量人工调研积累的，竞争对手难以快速复制。

#### 2. 体验壁垒

**专为"找动漫店"场景设计的地图UI**——不同于 Google Maps 的通用搜索、小红书的图文瀑布流，我们的产品从交互到视觉都围绕"在地图上快速找到、筛选、定位动漫店"这一垂直场景优化。7色品牌编码、卡片-标记联动、地区/品牌组合筛选等功能，在通用平台上无法实现。

#### 3. 成本壁垒

**近乎零成本的运营架构**——GitHub Pages免费托管 + OpenFreeMap免费瓦片 + 静态前端无服务器开销。这意味着我们可以在没有收入的情况下持续运营产品，而竞争对手通常需要承担服务器、API调用、CDN等持续成本。

#### 4. 速度壁垒

**3天完成MVP的极速迭代能力**——从Day 1的87家店铺、3个品牌，到Day 3的176家店铺、7个品牌，包含WebGL迁移、Android APK、SEO优化等重大技术升级。这种执行速度意味着我们能以极低成本快速验证想法，抢在竞争对手之前占领市场。

#### 5. 社区壁垒（规划中）

**未来用户生态将形成网络效应**——当用户开始贡献评价、照片和路线分享后，每个新用户都会让平台对所有用户更有价值。这种 UGC 网络效应是最强大的竞争壁垒之一。

## 6. 商业模式

### 6.1 收入来源

我们规划了**六大收入来源**，按实施时间线排列：

#### 收入来源一：Freemium 订阅（Phase 2启动）

| 层级 | 价格 | 功能 |
|------|------|------|
| 免费版 | ¥0 | 地图浏览、搜索筛选、基础店铺信息 |
| 高级版 | ¥30/月 或 ¥288/年 | 收藏夹同步、完整离线地图包、行程规划、路线优化、优先数据更新 |

- **目标转化率：** 3%-5% 的 MAU
- **参考案例：** 航旅纵横PRO、墨迹天气VIP等工具类应用付费模式

#### 收入来源二：店铺合作伙伴计划（Phase 3启动）

| 合作形式 | 月费 | 说明 |
|----------|------|------|
| 推荐置顶 | ¥500-2,000/月 | 搜索结果和地区页面优先展示 |
| 专属页面 | ¥1,000-3,000/月 | 店铺详情页定制内容（促销信息、库存亮点等） |
| 优惠券分发 | 按核销量分成 | 在地图中向用户推送店铺优惠券，按实际使用量结算 |

#### 收入来源三：联盟电商佣金（Phase 3启动）

- 为店铺在线商城提供导流链接（如骏河屋、Mandarake网店）
- 佣金比例 **3%-5%**
- 参考案例：日本乐天联盟、Amazon Associates

#### 收入来源四：精准广告（Phase 2启动）

| 广告类型 | CPM | 说明 |
|----------|-----|------|
| 旅行保险 | ¥50-80 | 赴日旅行场景高度相关 |
| 机票/酒店 | ¥30-60 | 旅行规划阶段精准触达 |
| Wi-Fi/SIM卡 | ¥40-70 | 出境刚需产品 |
| 动漫活动 | ¥20-40 | Comiket、AnimeJapan等 |

- **原则：** 仅投放与旅行/动漫高度相关的广告，不影响用户体验
- **位置：** 搜索结果底部、行程规划页面侧栏（非侵入式）

#### 收入来源五：数据与API授权（Phase 4启动）

- 向日本观光局、地方政府、旅行社授权脱敏后的旅行行为数据
- 提供 API 接口，允许第三方应用接入店铺数据
- 定价：按调用量或年度订阅

#### 收入来源六：B2B白标方案（Phase 4启动）

- 将地图引擎和店铺数据以白标形式授权给旅行社、OTA平台
- 场景：携程/飞猪的日本旅游频道内嵌动漫店铺地图
- 定价：年度授权费 + 按用量分成

### 6.2 定价策略

**核心原则：免费优先，价值驱动付费。**

```
免费用户 ──────────────────► 高级用户 ──────────────► 商业用户
(地图/搜索/筛选)            (收藏/离线/行程)          (API/白标/数据)
¥0                          ¥30/月                    定制报价
目标：最大化用户基数          目标：3-5% 转化            目标：高ARPU收入
```

- **免费层足够好用**，确保用户留存和口碑传播
- **付费层解决真实痛点**（离线地图在旅行中是刚需，行程规划节省大量时间）
- **合作伙伴定价**根据店铺规模和所在城市流量差异化定价

| **竞争风险** | Google Maps 或其他平台推出动漫店铺专项功能 | 低 | 高 | ① 持续深耕垂直领域，保持数据深度和中文体验的绝对优势 ② 建设用户社区形成网络效应 ③ 通用平台难以做到"品牌分类""中文描述"等垂直功能 |
| **数据风险** | 店铺搬迁/关闭导致数据过时 | 高 | 中 | ① 建立自动化数据检测和更新机制 ② 开放用户纠错入口，众包维护数据准确性 ③ 与店铺方建立直接数据通道 |
| **技术风险** | OpenFreeMap 服务中断或政策变更 | 低 | 中 | ① 技术架构已支持多瓦片源切换（MapTiler、Stadia Maps等备选） ② 核心数据本地化存储，不依赖单一外部服务 ③ 可自建瓦片服务器作为终极后备 |
| **变现风险** | 用户付费意愿低于预期 | 中 | 中 | ① 保持强大的免费版确保用户基数 ② 重心转向B2B收入（API授权/白标/店铺合作） ③ 广告收入作为保底 ④ 持续优化付费功能的价值感知 |
| **团队风险** | 创始人单点依赖，无法分身 | 高 | 高 | ① 代码完整文档化（GitHub开源，README详尽） ② 种子轮后首要任务招聘核心技术成员 ③ 建立标准化开发流程和知识库 |
| **市场风险** | 动漫旅游市场规模小于预估 | 低 | 中 | ① 产品架构可扩展至泛御宅族文化（游戏店、偶像店、Cosplay店等） ② 覆盖更多日本特色购物场景（药妆、免税店等） ③ 工具属性使其在小市场也能生存 |

### 风险矩阵

```
影响程度
  高 │  团队风险    政策风险
     │             竞争风险
     │
  中 │  变现风险    数据风险
     │  市场风险    技术风险
     │
  低 │
     └──────────────────────
        低    中    高
            发生概率
```

### 总体风险评估

当前产品的核心风险可控，原因如下：

1. **零成本运营模式**意味着即使增长放缓，产品也不会因资金枯竭而被迫关闭
2. **技术架构的灵活性**（多瓦片源、纯前端无后端依赖）降低了技术风险
3. **数据资产**随时间推移持续增值，形成越来越深的竞争壁垒
4. **市场方向可调整**——从动漫店扩展到泛文化购物、从日本扩展到全亚洲，战略空间充足
